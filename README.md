# EnigmaSql — розничный ETL: анонимизация и объединение таблиц

Читает Excel/CSV с продажами из нескольких магазинов, дедуплицирует товары между ними, анонимизирует данные, объединяет в одну таблицу и архивирует исходники.

---

## Быстрый старт

```bash
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt

python pipeline.py
```

Скрипт интерактивно попросит выбрать три папки:

- **исходники** — положи сюда входные файлы (`.xlsx`, `.xls`, `.csv`, `.tsv`)
- **архив** — куда переместятся исходники после обработки
- **результат** — куда сохранится итоговый `merged_YYYYMMDD.xlsx`

Папка выбирается номером из списка или Enter (оставить текущую). Выбор сохраняется в `config.json` — при следующем запуске можно жать Enter × 4.

---

## Что происходит

1. Читает все файлы из входной папки, распознаёт заголовки (русские/английские варианты → единый canon)
2. **Дедуплицирует товары** до анонимизации:
   - стадия A: точное совпадение по штрихкоду
   - стадия B: нечёткое по названию (TF-IDF + гейты: объём, оттенок, аромат, тип товара, редкие токены)
3. Анонимизирует: магазины/бренды/товары/компании → синтетические псевдонимы, метрики → смещены
4. Объединяет по общим столбцам, суммирует метрики по совпадающим строкам
5. Архивирует исходники

Маппинг кэшируется в `mappings.json` — одна и та же реальная сущность всегда получает одно фейковое имя.

## Файлы проекта

| Файл | Назначение |
|---|---|
| `pipeline.py` | ядро: чтение, матчинг, анонимизация, merge, CLI |
| `dagster_job.py` | Dagster ops/job/sensor |
| `producer.py` | отправить команду `/merge` в Kafka |
| `consumer.py` | читать уведомления из Kafka |
| `eval_matching.py` | замерить precision/recall матчинга на golden set |
| `docker-compose.yml` | локальный Kafka-брокер (KRaft) |

---

## Оценка качества матчинга

```bash
python eval_matching.py                     # порог 0.82 по умолчанию
python eval_matching.py --threshold 0.78    # другой порог
python eval_matching.py --sweep             # прогнать 0.70–0.92 разом
```

Нужен файл `golden_pairs.json`:

```json
[
  {"a": "Название А", "b": "Название Б", "label": "match"},
  {"a": "Название А", "b": "Название В", "label": "non-match"}
]
```

Приоритет при добавлении пар — товары одинакового объёма (разные размеры режет size-гейт и без golden set).

## Dagster + Kafka (событийный режим)

```bash
docker compose up -d           # поднять брокер
dagster dev -f dagster_job.py  # UI на localhost:3000, сенсор стартует автоматически

python consumer.py   # ждёт результата в etl-results
python producer.py   # вводишь /merge → сенсор подхватит и запустит job
```

Пути можно переопределить через `run_config.yaml` в UI:

```yaml
resources:
  paths:
    config:
      input_folder:   "data/raw"
      archive_folder: "data/archive"
      output_folder:  "data/output"
```
